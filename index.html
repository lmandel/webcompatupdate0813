But the mobile Web is broken

This results in a poor user experience in Firefox products

:(
  • Directly impacts the Firefox product experience
  • Not something over which Mozilla has direct control

Desktop vs mobile content

  • Poor expeience due to desktop vs mobile content
  • Content is hard to read
  • Content can be very difficult to interact with when features like hover over menus are in use

Basic HTML vs HTML5

  • Poor expeience due to simplified mobile content
  • Typically static HTML - non interactive content

4 issues

  1. User Agent (UA) detection
  2. Vendor (Webkit) specific CSS
  3. Functional DOM/JS issues
  4. Gecko limitations


Learn more about the issues from my Oct 2012 video update (4:00)
  • UA detection - not recognizing Firefox as mobile or not recognizing it as feature capable mobile
  • CSS - properties that Gecko doesn't understand - includes functional issues
  • DOM/JS - vendor specific implementations or quirks
  • Gecko - limitations in our own platform
